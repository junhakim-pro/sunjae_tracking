# 선재 이유식/분유/수면 기록 플랫폼 기획 초안

## 1. 제품 한 줄 정의

가족이 카카오톡 또는 텔레그램 챗봇에 자연스럽게 남긴 말과 사진 기록을 AI가 읽어 구조화하고, 아기의 분유/이유식/수면/메모를 한 화면에서 공유·통계·추적하는 육아 로그 플랫폼.

## 2. 문제 정의

현재 기록 방식은 다음과 같이 분산되어 있다.

- 낮 시간: 부모님이 수첩에 수기로 기록
- 이동 시점: 수첩 사진을 카카오톡으로 공유
- 저녁/아침: 보호자가 카카오톡 메시지로 직접 기록

이 방식의 문제는 다음과 같다.

- 기록 포맷이 제각각이라 일/주 단위 합산이 어렵다
- 사진 속 필기 기록은 나중에 검색/통계가 거의 불가능하다
- 가족이 같은 정보를 봐도 "오늘 총 얼마나 먹었는지" 즉시 파악하기 어렵다
- 수면/감기/컨디션 같은 맥락 데이터가 분리되어 있어 원인 분석이 어렵다

## 3. 해결 방향

핵심은 입력 도구를 새로 학습시키는 것이 아니라, 기존 대화 습관을 최대한 유지한 채 AI가 기록을 정규화하는 것이다.

예시:

- "10시 40분 분유 180 다 먹음"
- "이유식 소고기브로콜리 120 정도 먹고 20 남김"
- "1시 10분 낮잠 시작, 2시 기상"
- "콧물 좀 있음, 오늘 입맛 별로"
- 수첩 사진 업로드

AI는 위 입력을 구조화해 아래처럼 저장한다.

- 섭취 기록: 시간, 종류, 섭취량, 남긴 양, 장소, 입력자, 신뢰도
- 수면 기록: 시작/종료, 총 수면 시간
- 메모 기록: 감기, 컨디션, 특이사항

## 4. 주요 사용자

- 엄마/아빠: 실시간 입력, 요약 확인, 추세 확인
- 부모님: 가장 쉬운 방식으로 기록 입력
- 아기: 직접 사용자는 아니지만, 개인화 기준의 중심 객체

## 5. 핵심 가치

- 개떡같이 입력해도 기록이 남는다
- 오늘 총 섭취량과 다음 식사 타이밍이 즉시 보인다
- 사진/메시지/웹 입력이 모두 같은 로그로 합쳐진다
- 키/몸무게 기반 성장 퍼센타일과 섭취 추세를 같이 본다

## 6. MVP 범위

### 필수 기능

1. 메시지 기반 기록 입력
- 텍스트 자유 입력
- 사진 업로드 입력
- 입력 즉시 AI 파싱 후 구조화

2. 기록 카테고리
- 분유
- 이유식
- 수면
- 기타 메모

3. AI 응답
- 파싱 결과 확인
- 금일 권장 총량 대비 달성률
- 다음 식사 추천 시점
- 누락/애매한 부분 재질문

4. 웹 플랫폼
- 오늘 타임라인
- 일/주 통계
- 최근 키/몸무게
- 성장 퍼센타일 표시
- 메모/컨디션 이슈 확인

5. 가족 공유
- 동일 아기 프로필에 여러 보호자 참여
- 누가 어떤 기록을 남겼는지 표시

6. 비용 제어
- 이미지 인식은 하루 2회까지
- 그 외는 텍스트 중심 파싱

### MVP에서 일부러 제외할 것

- 의학적 진단
- 자동 질병 판단
- 복수 아기 고급 권한 체계
- 병원/예방접종/쇼핑 연동
- 카카오톡과 텔레그램 동시 정식 런칭

## 7. 추천 사용자 흐름

### A. 보호자 온보딩

1. 웹 또는 챗봇에서 가입
2. 아기 정보 입력
- 이름
- 생년월일
- 성별
- 최근 키/몸무게
- 기본 수유/이유식 목표량
3. 가족 초대 링크 공유

### B. 기록 입력

1. 사용자가 챗봇에 자유롭게 입력
2. AI가 구조화 JSON 초안 생성
3. 서버가 저장
4. 챗봇이 요약 응답

응답 예시:

> 14:20 분유 180ml로 기록했어요.
> 오늘 분유 목표 720ml 중 58% 달성했어요.
> 마지막 수유 기준으로 다음 수유는 약 3시간 뒤가 좋아 보여요.

### C. 사진 입력

1. 사용자가 수첩 사진 업로드
2. OCR + 멀티모달 파싱 수행
3. 후보 기록 여러 건 추출
4. 사용자가 "맞아요"만 누르면 일괄 저장

### D. 웹 확인

- 오늘 총 분유량 / 총 이유식량 / 총 수면시간
- 최근 7일 추세
- 마지막 키/몸무게 기준 퍼센타일
- 컨디션 메모와 섭취량 감소 상관관계

## 9. AI 설계 방향

### 9-1. 파싱 전략

AI는 "답변 생성"보다 "구조화 추출"에 집중해야 한다.

- 입력 문장 -> 정규 스키마로 변환
- 애매하면 추측 저장하지 않고 확인 질문
- 예: "조금 먹음"은 그대로 저장하지 말고 `amount_unknown` 처리

### 9-2. 비용 관리

- 텍스트 입력은 경량 모델 우선
- 이미지 입력만 멀티모달 모델 사용
- 이미지 분석은 아기별 하루 2회 제한
- 같은 사진 재업로드는 해시 비교로 중복 차단

### 9-3. 신뢰도 처리

- `parse_confidence`가 낮으면 저장 전 사용자 확인
- 예: 시간 누락, ml/g 단위 불명확, 식사 종류 불명확

## 10. 권장량과 피드백 설계

이 부분은 자동 의료 판단으로 보이지 않게 설계하는 것이 중요하다.

추천 방식:

- 기본은 보호자가 직접 설정한 일일 목표량 사용
- 보조적으로 월령/체중 기반 "참고 범위"를 표시
- 문구는 "의료 진단"이 아니라 "기록 기반 참고"로 표현

권장 UX:

- "오늘 목표 800ml 중 62% 달성"
- "최근 3일 평균보다 14% 적어요"
- "감기 메모가 있던 날에는 섭취량이 평균보다 낮았어요"

피해야 할 UX:

- "정상/비정상"
- "먹는 양이 부족합니다" 단정
- 의료적 확정 문구

## 11. 성장 퍼센타일 설계

플랫폼 상단에 다음 정보를 고정 표시하는 구성이 적절하다.

- 현재 월령
- 마지막 측정일
- 최근 몸무게 / 키
- 몸무게 퍼센타일
- 키 퍼센타일

이후 차트로 다음을 제공한다.

- 몸무게 퍼센타일 변화
- 키 퍼센타일 변화
- 섭취량 추세와 성장 추세 비교

실무적으로는 WHO 0-5세 성장 표준 또는 국내 공신력 있는 소아 성장 도표를 기준 테이블로 넣고 계산하면 된다.

## 12. 카카오톡 vs 텔레그램 비교

### 카카오톡 장점

- 부모님 세대에게 가장 익숙함
- 별도 앱 설치 장벽이 낮음
- 가족 공유 채널로 이미 사용 중일 가능성이 큼

### 카카오톡 단점

- 일반 개발자 관점에서 텔레그램보다 챗봇/양방향 자동화 구성이 더 제한적이고 비즈니스 설정 의존도가 높음
- 카카오톡 채널, 챗봇 관리자센터, 카카오싱크 등 여러 비즈니스 도구를 함께 맞춰야 하는 경우가 많음
- 빠르게 개인 프로젝트형 MVP를 만들 때 시행착오가 많을 수 있음

### 텔레그램 장점

- Bot API가 단순하고 공개 문서가 매우 명확함
- 자유 텍스트, 사진, 버튼, 웹앱(Mini App) 연동이 쉬움
- 봇 + 웹앱 결합 MVP 제작 속도가 빠름
- 운영/디버깅/확장성이 좋음

### 텔레그램 단점

- 부모님 세대에게는 새로운 앱 설치가 필요할 수 있음
- 한국 가족 커뮤니케이션 기본 채널로는 카카오톡보다 낯설 수 있음

### 권장 결론

1차 MVP는 텔레그램으로 만드는 것이 가장 현실적이다.

이유:

- 챗봇 입력 UX와 웹앱 연동을 빠르게 검증할 수 있음
- AI 파싱 정확도와 기록 구조 설계를 먼저 검증 가능
- 이후 실제 반응이 좋으면 카카오톡은 2단계로 확장 가능

실행 전략:

- 1단계: 텔레그램 챗봇 + 웹 플랫폼
- 2단계: 웹 직접 입력 + 카카오 로그인/카카오싱크 검토
- 3단계: 카카오톡 채널 챗봇 또는 대체 UX 검토

- 시간순 기록 카드

### 통계

- 최신 키/몸무게

- 가족 초대

## 15. 가장 중요한 제품 원칙

1. 입력은 자유롭게, 저장은 구조적으로
2. AI는 기록 보조자이지 의사처럼 말하지 않기
3. 가족 중 가장 디지털 친화적이지 않은 사용자도 입력 가능해야 함
4. "오늘 얼마나 먹었나"가 3초 안에 보여야 함

## 16. MVP 출시 체크리스트

- 자유 텍스트 20개 패턴에서 분유/이유식/수면 파싱 성공
- 사진 업로드 하루 2회 제한 동작
- 1일 총량 집계 정확성 검증
- 가족 2-3명 공동 기록 충돌 처리
- 수정 이력 남기기
- 목표량 대비 달성률 계산
- 다음 식사 추천 로직 검증
- 성장 퍼센타일 계산 표시
