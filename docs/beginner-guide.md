# 초보자용 진행 가이드

## 1. 이 프로젝트는 어떻게 진행하면 되나

결론부터 말하면, 꼭 GitHub에 올려야만 하는 건 아니다.

하지만 실제로 서비스를 계속 만들고 나중에 배포까지 하려면 GitHub를 쓰는 편이 훨씬 좋다.

이유는 아주 단순하다.

- 코드 백업이 된다
- 어디까지 만들었는지 기록이 남는다
- 배포 서비스(Vercel 등)와 연결이 쉽다
- 나중에 문제가 생겨도 이전 상태로 돌아가기 쉽다

즉, **필수는 아니지만 거의 표준에 가깝다**고 보면 된다.

## 2. 초보자 기준 추천 흐름

가장 쉬운 흐름은 아래다.

1. 이 폴더를 GitHub 저장소로 올린다
2. Vercel에 연결해서 웹앱을 배포한다
3. 환경변수만 넣는다
4. 텔레그램 webhook을 연결한다
5. 실제 텔레그램에서 메시지를 보내본다

## 3. 지금 이 프로젝트의 상태

이미 여기까지는 되어 있다.

- 웹 화면 있음
- DB 연결됨
- 웹에서 직접 기록 입력 가능
- 최근 7일 추세 표시
- 텔레그램 webhook 코드 있음
- DeepSeek / Qwen / OpenAI 중 골라 연결 가능
- 사진 하루 2회 제한 있음

즉, 지금부터 필요한 건 "새 기능을 더 만드는 것"보다 **배포하고 실제로 붙여보는 단계**다.

## 4. GitHub는 꼭 써야 하나

### 안 써도 되는 경우

- 혼자 잠깐 실험만 해볼 때
- 배포 없이 로컬에서만 볼 때

### 쓰는 게 좋은 경우

- 계속 기능을 붙일 계획이 있을 때
- 나중에 배포할 때
- 혹시 컴퓨터가 꼬이거나 파일이 날아갈까 걱정될 때

현재 상황에서는 **GitHub 사용을 권장**한다.

## 5. 내가 앞으로 어떻게 진행할 건가

앞으로는 아래 원칙으로 진행한다.

- 비용/토큰/외부 계정처럼 꼭 필요한 것만 확인
- 그 외 구현/정리/문서화는 내가 계속 진행
- 시각적으로 보여줄 가치가 있는 단계면 화면 기준으로 설명
- 가능한 한 사용자가 헷갈리지 않게 "다음 행동"이 보이게 정리

## 6. 다음에 사용자가 직접 해줘야 하는 것

아래는 내가 대신할 수 없고, 사용자 본인이 해야 하는 것들이다.

- GitHub 계정 로그인
- 텔레그램 봇 생성
- DeepSeek 또는 Qwen API 키 발급
- Vercel 로그인
- 배포 서비스에 환경변수 입력

나머지는 내가 계속 코드와 문서 기준으로 진행해도 된다.
