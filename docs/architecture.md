# 아키텍처 초안

## 목표

가족이 메신저에 자유 텍스트 또는 사진을 남기면 서버가 이를 구조화하고, 웹 대시보드에서 일일 로그와 추세를 볼 수 있게 한다.

## 권장 구조

### 1. 입력 계층

- 텔레그램 봇 webhook
- 웹 직접 입력 폼
- 추후 카카오 확장용 추상화된 `message-ingest` 계층

### 2. 처리 계층

- 원문 저장
- 이미지 사용 횟수 제한 검사
- AI 파싱 요청
- 스키마 검증
- 신뢰도 기반 후속 질문 생성

### 3. 저장 계층

- Baby
- Caregiver
- IntakeLog
- SleepLog
- NoteLog
- GrowthMeasurement
- RawMessage

### 4. 표시 계층

- 오늘 대시보드
- 타임라인
- 주간 통계
- 성장 추세

## 서버 흐름

1. 텔레그램이 `/api/telegram/webhook`으로 메시지를 전송
2. 서버가 원문과 메타데이터를 저장
3. 텍스트면 경량 파싱 모델, 이미지면 OCR/비전 경로로 분기
4. 결과를 `zod` 스키마로 검증
5. 낮은 신뢰도면 follow-up question 생성
6. 확정된 로그를 DB에 반영
7. 일일 합산 캐시를 갱신
8. 챗봇 응답으로 달성률과 다음 식사 추천 전송

## AI 경계 설계

- AI는 구조화만 담당
- 의료적 판단은 하지 않음
- 추천량은 보호자 입력 목표량이 우선
- 성장 퍼센타일은 기준 테이블 기반 계산으로 처리

## 프론트엔드 제안

- Next.js App Router
- 서버 컴포넌트로 요약 집계 로딩
- 차트는 추후 도입
- 모바일 우선 카드 UI
